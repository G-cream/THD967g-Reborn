1. 考虑到非原档967d与非原档967g含有相同数量的文件(1037个)，两个版本相去不远，有理由认为原档967g中并未新增加文件。
2. 通过对比可以看到非原档967d与非原档967g中只有j、w3a、w3i、w3t、w3u不一致，那么只要在967d原档中依次跟踪修改j、w3a、w3t、w3u，同时修改地图属性，就可以将967d原档升级为967g原档。
3. 在原档的复原中，有些数据是无法复原的，比如：
    a. j中的部分注释以及部分代码里的空格或缩进。
    b. w3a、w3t、w3u文件虽然可以确保字节顺序完全一致但是无法保证所有的数据结构结尾标识是一致的，虽然这并不影响数据结构本身的意义。
    c. w3i文件是无法保持一致的，但地图属性中其实只存在地图名字的修改，所以修改地图名字，保持意义一致即可。
4. 地图中所有的明改暗改都进行了还原，具体参考“actual changes from 967d - 967g.txt”
5. 如有修改上的遗漏，应参照“d-g文件用途.txt”重新对文件进行跟踪对比后修改。
